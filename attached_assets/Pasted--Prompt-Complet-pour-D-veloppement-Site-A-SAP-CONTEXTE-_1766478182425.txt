## 💳 SYSTÈME DE BOOKING

**Workflow:**
1. User parle à Agent IA → exprime besoin de RDV
2. Agent propose: "Je peux vous réserver un RDV Google Meet ou Teams. Quelle est votre préférence ?"
3. User choisit type + créneau souhaité
4. Agent affiche calendrier disponibilités (Cal.com embed)
5. User sélectionne créneau
6. Formulaire: nom, email, phone, company (optionnel)
7. Confirmation → création événement:
   - Google Meet: via Google Calendar API (meeting link auto)
   - Teams: via Microsoft Graph API (meeting link auto)
8. Email confirmation (Resend) avec lien meeting + ics
9. Reminders: 24h avant (email), 1h avant (email + SMS si phone fourni)

**Backend Booking:**
- Table `bookings` (Prisma schema fourni)
- Intégration Cal.com (webhook événements)
- Google Calendar API OAuth2
- Microsoft Graph API OAuth2
- Gestion timezone (Africa/Dakar par défaut, configurable)

## 💰 SYSTÈME DE PAIEMENT

**Stripe Integration:**
- Checkout Session pour formations
- Webhook `/api/webhooks/stripe` (signature verification)
- Événements:
  - `checkout.session.completed` → créer Enrollment ACTIVE
  - `payment_intent.succeeded` → envoyer email confirmation + accès LMS
  - `payment_intent.failed` → notifier échec

**Flow:**
1. User clique "S'inscrire" sur formation
2. Création Stripe Checkout Session
3. Redirection vers Stripe hosted checkout
4. Paiement → webhook → Enrollment créé
5. Email confirmation avec lien accès LMS
6. Dashboard étudiant accessible

**Support Mobile Money Afrique:**
- Stripe supporte Orange Money, MTN Money via partenaires
- Configurer dans Stripe Dashboard
